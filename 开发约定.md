# 原型页面开发约定

## PageLoader 与脚本位置

本系统通过 **PageLoader**（main.html）动态加载子页面：只将子页面中的 **`.content-wrapper`** 节点注入到 `#content-container`，并**仅执行该节点内部的 `<script>`**。

因此：

- **所有页面内联 `<script>` 必须写在 `.content-wrapper` 内部**（即放在包裹主内容区的 `<div class="content-wrapper">` 的闭合 `</div>` 之前）。
- 若将 `<script>` 写在 `.content-wrapper` 的闭合 `</div>` 之后，通过 main 进入该页时脚本不会执行，导致按钮事件、弹窗等交互失效。

### 正确结构示例

```html
<body>
    <div class="content-wrapper">
        <!-- 面包屑、筛选、表格、弹窗等 -->
        <div id="some-modal" class="modal-overlay">...</div>

        <script>
            // 页面逻辑写在这里，确保被 PageLoader 执行
        </script>
    </div>
</body>
```

### 错误结构（脚本不会执行）

```html
<body>
    <div class="content-wrapper">
        ...
    </div>
    <script>
        // 在 content-wrapper 外部，PageLoader 不会执行
    </script>
</body>
```

新增或修改被菜单/PageLoader 打开的页面时，请始终检查 `<script>` 是否在 `.content-wrapper` 内部。

## 主内容区宽度与排版

主内容区（右侧内容区）宽度由布局决定，**不要人为制造右侧大片空白**。

- **列表页、详情页、表单页**：卡片、表格、表单等内容应**占满主内容区可用宽度**，不要给卡片或内容容器加 `max-width` 把内容挤在左侧、右侧留一大块空白。
- 若确需限制行宽（如长段落可读性），再单独对**该段落**设 `max-width`，且避免整页内容都缩在一侧。
- 新增或改版页面时，检查在不同宽度下是否出现「右边一大片空白无内容」的情况，如有则去掉不必要的宽度限制。
